# React Context & Custom Hooks

## Custom Hooks

### 1. What are Custom Hooks?

A **custom hook** is a **JavaScript function that uses React hooks**
(`useState`, `useEffect`, etc.) to **encapsulate reusable logic**.

Custom hooks allow you to **reuse stateful logic between components**.

A custom hook:

- is just a **function**
- **must start with `use`**
- can call **other hooks**

Example names:

    useFetch
    useLocalStorage
    useWindowSize

---

### 2. Why do we use Custom Hooks?

When building React apps, you often repeat the same logic in multiple
components.

Example problems:

- fetching data
- handling form input
- listening to window resize
- managing localStorage
- handling authentication state

Instead of copying the same code into many components, we **extract it
into a custom hook**.

#### Without custom hook

Two components might repeat the same logic:

```jsx
useEffect(() => {
  fetch("/api/users")
    .then((res) => res.json())
    .then(setUsers);
}, []);
```

#### With custom hook

```jsx
const users = useUsers();
```

Benefits:

- **Code reuse**
- **Cleaner components**
- **Separation of concerns**
- **Easier testing**

Components focus on **UI**, hooks handle **logic**.

---

### 3. How do we write a Custom Hook?

A custom hook is simply a function that uses hooks.

#### Example: `useCounter`

```jsx
import { useState } from "react";

function useCounter(initialValue = 0) {
  const [count, setCount] = useState(initialValue);

  function increment() {
    setCount((c) => c + 1);
  }

  function decrement() {
    setCount((c) => c - 1);
  }

  return { count, increment, decrement };
}
```

---

### 4. Using the Custom Hook

Now any component can use it:

```jsx
function Counter() {
  const { count, increment, decrement } = useCounter(0);

  return (
    <div>
      <p>{count}</p>
      <button onClick={increment}>+</button>
      <button onClick={decrement}>-</button>
    </div>
  );
}
```

Important:

Each component gets **its own independent state**.

---

### 5. Custom Hook with Effects Example

Example: **Window width**

```jsx
import { useState, useEffect } from "react";

function useWindowWidth() {
  const [width, setWidth] = useState(window.innerWidth);

  useEffect(() => {
    function handleResize() {
      setWidth(window.innerWidth);
    }

    window.addEventListener("resize", handleResize);

    return () => {
      window.removeEventListener("resize", handleResize);
    };
  }, []);

  return width;
}
```

Usage:

```jsx
function App() {
  const width = useWindowWidth();

  return <p>Window width: {width}</p>;
}
```

---

### 6. Rules for Custom Hooks

Custom hooks follow the **same rules as React hooks**:

1.  **Must start with `use`**
2.  **Only call hooks at the top level**
3.  **Only call hooks from React functions or other hooks**

Correct:

```jsx
function useSomething() {
  const [value, setValue] = useState(0);
}
```

Wrong:

```jsx
function something() {
  const [value, setValue] = useState(0); // not a hook
}
```

---

### 7. Mental Model

Think of custom hooks as:

> **Reusable logic modules for React components**

Components handle:

- UI
- rendering

Hooks handle:

- logic
- state
- side effects

---

### 8. Typical Real-World Hooks

Examples you often see in real projects:

    useFetch()
    useLocalStorage()
    useDebounce()
    useAuth()
    useMediaQuery()
    useForm()
    useToggle()

---

    src

---

✅ **Key takeaway**

Custom hooks allow you to:

- **extract reusable state logic**
- **keep components clean**
- **share logic across many components**

## React Context (with Provider + Custom Hook Pattern)

This summary shows a **modern React Context setup** where:

- the **Context** is defined in one file
- the **Provider** handles state logic in another file
- a **custom hook** provides convenient access to the context

This pattern keeps the code clean and avoids warnings that sometimes
appear in newer **Vite / React setups** when context, provider, and
hooks live in the same file.

---

### 1. What is React Context?

**React Context** allows components to **share data without passing
props through every level of the component tree**.

Normally, data flows like this:

    Parent → Child → Grandchild

With Context, any component inside a subtree can directly access shared
data.

Typical context data:

- language
- theme (dark / light)
- authentication state
- user data
- global configuration

---

### 2. Why do we use React Context?

Imagine this component tree:

    App
     ├ Navbar
     │   └ LanguageSwitcher
     └ Page
         └ CommentSection
             └ CommentForm

If **language** is needed everywhere, we would otherwise pass props
through many levels:

    <App language={language}>
      <Navbar language={language}>
        <LanguageSwitcher language={language} />
      </Navbar>
    </App>

This is called **prop drilling**.

React Context allows us to **provide the language once and use it
anywhere inside the tree**.

Benefits:

- avoids prop drilling
- cleaner components
- centralized state
- easier maintenance

---

### 3. Recommended File Structure

In many modern projects we separate Context, Provider and Hook.

    src
      context
        LanguageContext.js
        LanguageProvider.jsx
        useLanguage.js

This setups avoids warnings regarding the "Hot Reload" feature of Vite/React (=> If a file exports a component, it should be the only export in that file).

---

### 4. Step 1 --- Create the Context

File: **LanguageContext.js**

```jsx
import { createContext } from "react";

export const LanguageContext = createContext(null);
```

The context object is just the **data channel**.\
It does not contain state logic.

---

### 5. Step 2 --- Create the Provider

File: **LanguageProvider.jsx**

The provider contains the **state and logic**.

```jsx
import { useState } from "react";
import { LanguageContext } from "./LanguageContext";

export default function LanguageProvider({ children }) {
  const [language, setLanguage] = useState("en");

  function changeLanguage(lang) {
    setLanguage(lang);
  }

  const value = {
    language,
    changeLanguage,
  };

  return (
    <LanguageContext.Provider value={value}>
      {children}
    </LanguageContext.Provider>
  );
}
```

The provider wraps the part of the app that should access the context (`children` prop).

---

### 6. Step 3 --- Create a Custom Hook

File: **useLanguage.js**

A custom hook makes accessing the context easier.

```jsx
import { useContext } from "react";
import { LanguageContext } from "./LanguageContext";

export function useLanguage() {
  return useContext(LanguageContext);
}
```

Now components do not need to import the context directly.

---

### 7. Using the Provider

In your entry point (usually `main.jsx`), wrap the Context Provider around your `<App>` component:

```jsx
import LanguageProvider from "./context/LanguageProvider";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <LanguageProvider>
      <App />
    </LanguageProvider>
  </StrictMode>,
);
```

Now all components inside `<App />` can access the language context.

---

# 8. Using the Custom Hook

```jsx
import { useLanguage } from "../context/useLanguage";

function LanguageSwitcher() {
  const { language, changeLanguage } = useLanguage();

  return (
    <div>
      <p>Current language: {language}</p>

      <button onClick={() => changeLanguage("en")}>English</button>
      <button onClick={() => changeLanguage("de")}>Deutsch</button>
    </div>
  );
}
```

The hook internally calls:

    useContext(LanguageContext)

but the component does not need to know about the context
implementation.

---

# 9. Mental Model

Think of Context like a **shared data channel**.

    Provider
       ↓
    React Component Tree
       ↓
    useContext / custom hook

Components subscribe to the data provided by the provider.

---

# 10. Key Takeaway

React Context helps you:

- avoid **prop drilling**
- share **global state**
- keep components clean

A common modern pattern is:

    Context
    Provider
    Custom Hook

which keeps your React code **structured and scalable**.
