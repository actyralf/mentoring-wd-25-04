## The `useReducer` hook

## What is `useReducer`?

`useReducer` is a React Hook for managing **complex state logic**.

It is a good alternative to `useState` when:

- state has multiple related values
- updates follow clear rules
- you want more structure
- different user actions can change the state in different ways

---

## Basic Idea

With `useState`, we often set the next state directly.

With `useReducer`, we instead:

1. describe **what happened** with an **action**
2. let the **reducer function** decide how the state changes

---

## The Syntax

```tsx
const [state, dispatch] = useReducer(reducer, initialState);
```

- `state` = the current state
- `dispatch` = function used to send an action
- `reducer` = function that receives current state and action, then returns new state

---

## Core Concepts

### 1. State Type

In TypeScript, we often define the shape of the state first.

```ts
type State = {
  count: number;
};
```

---

### 2. Action Type

Actions are usually objects with a `type`.

Sometimes they also include extra data.

```ts
type Action =
  | { type: "increment" }
  | { type: "decrement" }
  | { type: "add"; payload: number };
```

This is called a **union type**.

It tells TypeScript exactly which actions are allowed.

---

### 3. Reducer Function

The reducer gets:

- the current `state`
- an `action`

and returns the new state.

```ts
function reducer(state: State, action: Action): State {
  switch (action.type) {
    case "increment":
      return { count: state.count + 1 };

    case "decrement":
      return { count: state.count - 1 };

    case "add":
      return { count: state.count + action.payload };

    default:
      return state;
  }
}
```

---

## Complete TypeScript Demo

```tsx
import { useReducer } from "react";

type State = {
  count: number;
};

type Action =
  | { type: "increment" }
  | { type: "decrement" }
  | { type: "add"; payload: number };

const initialState: State = {
  count: 0,
};

function reducer(state: State, action: Action): State {
  switch (action.type) {
    case "increment":
      return { count: state.count + 1 };

    case "decrement":
      return { count: state.count - 1 };

    case "add":
      return { count: state.count + action.payload };

    default:
      return state;
  }
}

export default function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState);

  return (
    <div>
      <p>{state.count}</p>

      <button onClick={() => dispatch({ type: "increment" })}>+1</button>

      <button onClick={() => dispatch({ type: "decrement" })}>-1</button>

      <button onClick={() => dispatch({ type: "add", payload: 5 })}>+5</button>
    </div>
  );
}
```

---

## What Happens Here?

When the user clicks a button:

1. `dispatch(...)` is called
2. the action object is sent to the reducer
3. the reducer checks `action.type`
4. the reducer returns a new state object
5. React re-renders the component

---

## Why TypeScript Is Helpful Here

TypeScript improves `useReducer` because it:

- checks that state has the correct structure
- checks that only valid actions are dispatched
- checks that required payload data is present
- gives better autocomplete and error messages

Example:

```ts
dispatch({ type: "add", payload: 5 }); // correct
dispatch({ type: "increment" }); // correct
dispatch({ type: "add" }); // TypeScript error
dispatch({ type: "reset" }); // TypeScript error
```

---

## `useState` vs `useReducer`

| `useState`            | `useReducer`                |
| --------------------- | --------------------------- |
| best for simple state | best for more complex state |
| update directly       | dispatch actions            |
| quick and easy        | more structured             |
| less code at first    | easier to scale later       |

---

## When Should You Use `useReducer`?

`useReducer` is useful when:

- several values belong together
- many different actions can update the same state
- the logic becomes hard to manage with many `setState` calls
- you want predictable state transitions

Typical examples:

- counters with multiple actions
- forms
- filters
- shopping carts
- game state
- loading / success / error UI states

---

## Important Rule

Do **not** mutate the state directly.

Wrong:

```ts
state.count++;
return state;
```

Correct:

```ts
return { count: state.count + 1 };
```

Always return a **new object**.

---

## Short Mental Model

`useReducer` means:

- **state** = current data
- **action** = what happened
- **reducer** = rule for updating the state

So instead of saying:

> â€śSet the state to this value.â€ť

you say:

> â€śThis action happened. Please calculate the next state.â€ť

---

## Summary

- `useReducer` is a Hook for **complex state logic**
- it uses a **reducer function** and **actions**
- `dispatch` sends an action to the reducer
- the reducer returns the new state
- TypeScript makes the pattern safer and easier to understand

## TypeScript Generics

Generics allow you to write **flexible, reusable functions and components** that work with different types.

đź‘‰ Instead of fixing a type (e.g. `string`), you use a **placeholder type**.

---

## Why do we need Generics?

Without generics:

```ts
function identity(value: any) {
  return value;
}
```

âťŚ Problem:

- no type safety
- TypeScript cannot help you

With generics:

```ts
function identity<T>(value: T): T {
  return value;
}
```

âś… Benefit:

- keeps type information
- safer and more predictable

---

## Basic Syntax

```ts
function myFunction<T>(param: T): T {
  return param;
}
```

- `T` = type placeholder (you can name it anything)
- Type is inferred automatically in most cases

---

## Example 1: Identity Function

```ts
function identity<T>(value: T): T {
  return value;
}

const result1 = identity("Hello"); // string
const result2 = identity(42); // number
```

---

## Example 2: Working with Arrays

```ts
function getFirst<T>(arr: T[]): T {
  return arr[0];
}

const firstNumber = getFirst([1, 2, 3]); // number
const firstString = getFirst(["a", "b"]); // string
```

---

## Example 3: Generic Types

You can also use generics with types:

```ts
type ApiResponse<T> = {
  data: T;
  success: boolean;
};
```

Usage:

```ts
type User = {
  name: string;
  age: number;
};

const response: ApiResponse<User> = {
  data: { name: "Anna", age: 30 },
  success: true,
};
```

---

## Example 4: React Example

```tsx
type SelectProps<T> = {
  items: T[];
  renderItem: (item: T) => React.ReactNode;
};

function Select<T>({ items, renderItem }: SelectProps<T>) {
  return (
    <ul>
      {items.map((item, index) => (
        <li key={index}>{renderItem(item)}</li>
      ))}
    </ul>
  );
}
```

Usage:

```tsx
<Select
  items={[1, 2, 3]}
  renderItem={(item) => <span>{item}</span>}
/>

<Select
  items={[{ name: "Anna" }]}
  renderItem={(item) => <span>{item.name}</span>}
/>
```

---

## Example 5: Constraining Generics

Sometimes we want to restrict what `T` can be.

```ts
function logLength<T extends { length: number }>(value: T): void {
  console.log(value.length);
}

logLength("Hello"); // OK
logLength([1, 2, 3]); // OK
logLength(123); // âťŚ Error
```

---

## How Type Inference Works

Usually you donâ€™t need to specify the type manually:

```ts
identity("Hello"); // TypeScript knows T = string
```

But you can do it explicitly:

```ts
identity<string>("Hello");
```

---

## When Should You Use Generics?

Use generics when:

- you want to reuse logic for different types
- the type is not known in advance
- you want **type safety + flexibility**

---

## Common Use Cases

- API responses
- reusable functions
- utility functions (map, filter, etc.)
- React components (lists, selects, tables)
- libraries and frameworks
