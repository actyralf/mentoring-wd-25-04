# useEffect in React

## What is `useEffect`?

`useEffect` is a **React Hook used to run side effects in function
components**.

A **side effect** is anything that interacts with the outside world or
happens **after rendering**, for example:

- Fetching data from an API\
- Setting timers (`setTimeout`, `setInterval`)\
- Subscribing to events\
- Manipulating the DOM\
- Logging or analytics

React renders the UI first, and then runs the effect.

---

## Basic Syntax

```jsx
import { useEffect } from "react";

useEffect(() => {
  // code that should run after render
}, []);
```

The hook takes **two arguments**:

1.  **Effect function** → what should run
2.  **Dependency array** → when it should run

---

## Common Patterns

### 1. Run once when the component mounts

```jsx
useEffect(() => {
  console.log("Component mounted");
}, []);
```

- The empty array `[]` means **run only once**
- Equivalent to `componentDidMount` in class components

Typical use case: **load data from an API**

```jsx
useEffect(() => {
  fetch("/api/users")
    .then((res) => res.json())
    .then((data) => setUsers(data));
}, []);
```

---

### 2. Run when a value changes

```jsx
useEffect(() => {
  console.log("Count changed:", count);
}, [count]);
```

- The effect runs whenever `count` changes.

You can include multiple dependencies:

```jsx
useEffect(() => {
  console.log("User or filter changed");
}, [userId, filter]);
```

---

### 3. Run after every render

```jsx
useEffect(() => {
  console.log("Rendered");
});
```

- No dependency array → runs **after every render**

This is **rarely needed**.

---

## Cleanup Functions

Some effects need cleanup (for example timers or subscriptions).

You can return a function from the effect:

```jsx
useEffect(() => {
  const id = setInterval(() => {
    console.log("tick");
  }, 1000);

  return () => {
    clearInterval(id);
  };
}, []);
```

The cleanup runs:

- when the component **unmounts**
- before the effect runs again

---

## Example Component

```jsx
import { useEffect, useState } from "react";

function Users() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetch("https://randomuser.me/api/?results=5")
      .then((res) => res.json())
      .then((data) => setUsers(data.results));
  }, []);

  return (
    <ul>
      {users.map((user) => (
        <li key={user.login.uuid}>
          {user.name.first} {user.name.last}
        </li>
      ))}
    </ul>
  );
}
```

This component:

1.  renders
2.  `useEffect` fetches users
3.  state updates
4.  component re-renders with the data

## Common Beginner Mistakes

### 1. Missing dependencies

```jsx
useEffect(() => {
  console.log(count);
}, []); // ❌ count missing
```

React's ESLint rules usually warn about this.

---

### 2. Using effects for things that don't need them

You **don't need `useEffect`** for:

- calculating derived values
- updating UI state from other state

Example:

```jsx
// ❌ unnecessary effect
useEffect(() => {
  setFullName(firstName + " " + lastName);
}, [firstName, lastName]);
```

Better:

```jsx
const fullName = firstName + " " + lastName;
```

---

## Mental Model

Think of `useEffect` as:

> **"Run this code after React has updated the UI."**

---

## Summary

`useEffect` is used for:

- data fetching
- timers
- subscriptions
- interacting with external systems

Key rule:

> **Effects synchronize your component with the outside world.**
